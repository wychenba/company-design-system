**Q3 self-verify mechanism propose**

Best verification: add a controlled Storybook probe story for PeoplePicker stack, not DOM-only clicks as the primary test.

Use a test-only story that exposes a tiny harness on `window`, e.g. `window.__peoplePickerRaceProbe.setValue([...])`, `clear()`, `selectAll()`, and `measure()`. Then Playwright can run exact state transitions:
- Path A: `setValue([p1]) -> ... -> setValue([p1..p6])`, awaiting two animation frames after each step.
- Path B: `setValue([p1..p6]) -> setValue([]) -> setValue([p1..p6])`, then await two animation frames.
- Assert same `tagArea.clientWidth`, same visible avatar count, same `+N`, and no hidden-state divergence.

DOM `dispatchEvent` can help, but it is less precise because Radix/cmdk/Popover event ordering, focus guards, and portal hit testing become part of the test. It verifies the user path, but it is noisy for root-cause isolation.

The existing probe script already aims at the right comparison: it measures visible avatar count and chip text for “逐個 click” vs “取消全選→再全選” in [peoplepicker-twopath-race-probe.mjs](/Users/chenqiren/Library/CloudStorage/GoogleDrive-qijenchen@gmail.com/我的雲端硬碟/my-project/scripts/peoplepicker-twopath-race-probe.mjs:32). I would keep that as an end-to-end regression, but add the controlled story/window setter as the deterministic root-cause test.
