**Verdict**

Ant Design 沒有單一跨元件 ordering canonical。

最常見 pattern 是「bulk operation 依目前 source/render data order 產生 keys」，但常搭配「保留既有 selection order，再 append 新 bulk keys」：Transfer、Table rowSelection 都是這個方向。Checkbox.Group 更強，最後直接 sort 成 `options` source order。Select/TreeSelect 的非 bulk click 則明確是 click append。

對 PeoplePicker 的 `Select All` 場景，最貼近的是 Transfer / Table rowSelection：bulk select 取可見/可選 options 的 source order，並和既有 selected 合併；如果要模仿 Checkbox.Group，則是回傳整體 options source order。
